# قائمة مراجعة مراجعة المحتوى العربي

## نظرة عامة

استخدم هذه القائمة للمراجعة النهائية لأي محتوى عربي في التطبيقات والمواقع.

---

## القسم الأول: الصوت والنبرة

### ✅ الصوت

- [ ] الصوت متسق مع هوية العلامة التجارية
- [ ] النبرة مناسبة للسياق (إيجابي/سلبي/محايد)
- [ ] اللغة إيجابية وليست سلبية
- [ ] غير رسمية وليست رسمية جداً

### ✅ النبرة

- [ ] تتناسب مع مشاعر المستخدم المتوقعة
- [ ] تستخدم الأبعاد الأربعة بشكل صحيح
- [ ] في المواقف السلبية: صادقة ومباشرة
- [ ] في المواقف الإيجابية: مشجعة ومُقدِّرة

---

## القسم الثاني: الوضوح والإيجاز

### ✅ الوضوح

- [ ] كل جملة واضحة المعنى
- [ ] لا توجد مصطلحات تقنية غير ضرورية
- [ ] الكلمات مألوفة للمستخدم العادي
- [ ] لا توجد缩写 غير مفهومة

### ✅ الإيجاز

- [ ] لا كلمات زائدة
- [ ] كل كلمة تخدم غرضاً
- [ ] الجمل قصيرة ( أقل من 20 كلمة)
- [ ] العناوين مختصرة وواضحة

---

## القسم الثالث: قواعد الكتابة

### ✅ علامات الترقيم

- [ ] لا نقاط زائدة في نهاية العناوين
- [ ] فاصلة بين الجمل المرتبطة
- [ ] علامات الترقيم العربية الصحيحة
- [ ] لا use English punctuation

### ✅ الأرقام

- [ ] الأرقام العربية في النصوص العربية (١٢٣)
- [ ] الأرقام الإنجليزية في السياقات التقنية (123)
- [ ] الفاصلة الصحيحة للأرقام الكبيرة (1,200)
- [ ] التاريخ بالتاريخ العربي الصحيح

### ✅ الحالة النحوية

- [ ] المبني للمعلوم (وليس للمجهول)
- [ ] جمل فعلية (وليس اسمية زائدة)
- [ ] أدوات الربط موجودة
- [ ] الضمائر صحيحة

---

## القسم الرابع: عناصر الواجهة

### ✅ الأزرار

- [ ] فعل + اسم (أضف كتاباً)
- [ ] واضحة ومحددة
- [ ] تختلف عن بعضها (أساسية/ثانوية)
- [ ] سياق التحول واضح

### ✅ رسائل الخطأ

- [ ] توضح المشكلة
- [ ] تقدم حلاً
- [ ] لاBlame المستخدم
- [ ] لغة طبيعية وليس تقنية

### ✅ رسائل النجاح

- [ ] تؤكد الإجراء
- [ ] تعرض الخطوة التالية
- [ ] تضيف قيمة
- [ ] إيجابية ومُشجِّعة

### ✅ رسائل التأكيد

- [ ] سؤال واضح
- [ ] توضح العواقب
- [ ] أزرار غير قابلة للخطأ
- [ ] حالة إلغاء الحساب: "غير مهتم"

---

## القسم الخامس: الاتساق

### ✅ المصطلحات

- [ ] نفس الكلمة لنفس العنصر في كل مكان
- [ ] اسم العلامة التجارية متسق
- [ ] المصطلحات التقنية موحدة

### ✅ التنسيق

- [ ] نفس الأسلوب في كل الصفحة
- [ ] العناوين بنفس المستوى
- [ ] الأزرار بنفس الأسلوب

### ✅ RTL

- [ ] الواجهة من اليمين لليسار
- [ ] الأرقام صحيحة الاتجاه
- [ ] لا مشاكل في المحاذاة

---

## القسم السادس: إمكانية الوصول

### ✅ الوضوح

- [ ] يمكن فهمه بدون سياق
- [ ] لغة مناسبة للجميع
- [ ] لا اختصارات غير مألوفة

### ✅ الملاحة

- [ ] الترتيب منطقي
- [ ] الخطوة التالية واضحة
- [ ] لاLoss of context

### ✅ التقنية

- [ ] متوافق مع قارئات الشاشة
- [ ] النصوص البديلة موجودة
- [ ] التباين كافٍ

---

## القسم السابع: الأخطاء الشائعة

### ✅ المراجعة

- [ ] لا "خطأ" أو "فشل" في بداية الرسائل
- [ ] لا كلمات سلبية بدون داعٍ
- [ ] لا رموز تعبيرية غير ضرورية
- [ ] لا "انقر هنا"

### ✅ الصياغة

- [ ] لا جمل ناقصة
- [ ] أدوات الربط موجودة
- [ ] الجمل ليست طويلة جداً
- [ ] لا تكرار غير ضروري

---

## القسم الثامن: سياقات خاصة

### ✅ نماذج التسجيل

- [ ] التسميات واضحة
- [ ] الحقول المطلوبة محددة
- [ ] أزرار العمل واضحة
- [ ] روابط المساعدة موجودة

### ✅ المدفوعات

- [ ] المبالغ واضحة
- [ ] العملات صحيحة
- [ ] ملخص الطلب واضح
- [ ] أزرار التأكيد واضحة

### ✅ الإشعارات

- [ ] مختصرة وواضحة
- [ ] الأيقونة مناسبة
- [ ] الإجراء واضح

---

## قائمة سريعة للمراجعة

```
□ هل يمكن فهم النص بدون أي سياق؟
□ هل هو إيجابي ومفيد؟
□ هل هو مختصر قدر الإمكان؟
□ هل المصطلحات متسقة؟
□ هل الأفعال صحيحة؟
□ هل علامات الترقيم صحيحة؟
□ هل RTL صحيح؟
□ هل الأزرار واضحة؟
□ هل رسائل الخطأ مفيدة؟
□ هل رسائل النجاح مشجعة؟
```
